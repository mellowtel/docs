رحنا نستبدلو `configuration_key` القديم بمعرف جديد يسمى **معرف التكامل**. إذا كنت تستعمل Mellowtel، لازمك تحدث السلسلة اللي تمررها لـ SDK.

## ملخص

- كل تكامل الآن عندو **معرف تكامل** خاص بيه، يبدأ بـ `intgr-`.
- الـ `configuration_key` القديم مازال يخدم اليوم، بصح رح يتوقف.
- التغيير الوحيد في الكود هو السلسلة اللي تمررها لـ `new Mellowtel(...)`.

## مفتاح التكوين مقابل معرف التكامل

| | مفتاح التكوين (قديم) | معرف التكامل (جديد) |
|---|---|---|
| النطاق | مفتاح واحد لكل حساب | معرف واحد لكل تكامل |
| الشكل | سلسلة غير شفافة | `intgr-XXXXXXXXXX` |
| الإحصائيات | مجمعة عبر جميع التطبيقات | لكل تطبيق |

## علاش رحنا نديرو هاد التغيير

Mellowtel الآن يخدم عبر إضافات المتصفح، تطبيقات سطح المكتب، تطبيقات الهاتف، والمواقع. بمفتاح واحد مشترك، الفرق ماقدروش يعرفو أي سطح كان فعلاً يجيب الأرباح.

معرف تكامل واحد لكل تطبيق يحل المشكلة. منين تهاجر، تفتح الإحصائيات لكل تطبيق، الأرباح لكل تطبيق، وميزات لوحة القيادة اللي رح نبنيوها باش تعاونك تنمي كل تطبيق بشكل مستقل.

## واش لازمك تبدل

غير السلسلة اللي تمررها لـ `new Mellowtel(...)`.

```javascript
// قبل
const mellowtel = new Mellowtel("<your_configuration_key>");

// بعد
const mellowtel = new Mellowtel("intgr-MG8tHhnkDN");
```

كل شيء آخر يبقى كيف كيف.

## كيفاش تجيب معرف التكامل

1. افتح [صفحة التكاملات](https://www.mellowtel.com/dashboard/integrations) في لوحة القيادة.
2. أنشئ تكامل واحد لكل تطبيق، واختار النوع المناسب (إضافة، سطح المكتب، الهاتف، الموقع).
3. انسخ معرف التكامل والصقه في تهيئة الـ SDK.

## التوافق مع الإصدارات القديمة

الكود الحالي يبقى يخدم. رح نعلن عن تاريخ التوقف مع الكثير من الوقت قبل ما نحيدو المفتاح القديم. الهجرة في أقرب وقت تفتح الميزات لكل تطبيق، يعني ماكانش علاش تستنى.
