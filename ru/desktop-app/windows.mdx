Интегрируйте Mellowtel в ваше Windows desktop приложение, чтобы пользователи могли делиться неиспользованной интернет-полосой в обмен на награды или премиум-функции.

<Warning>
  **Согласие пользователя обязательно.** SDK выбросит `InvalidOperationException`, если вы вызовете `StartAsync()` без согласия пользователя.
</Warning>

## Предварительные условия

- Аккаунт Mellowtel и **Integration ID** из [панели управления](https://www.mellowtel.com/mellowtel-auth/). Каждое desktop приложение получает уникальный Integration ID.
- .NET 10 SDK
- Windows 10 или Windows 11
- .NET desktop приложение (Console, WPF или Windows Forms)

<Note>
  Хотите увидеть полную, реальную интеграцию перед тем, как внедрить это в свое приложение? Пример [mellowtel-pomodoro-windows](https://github.com/mellowtel-inc/mellowtel-pomodoro-windows) для WPF показывает полный поток от начала до конца: диалог согласия, переключатель настроек и жизненный цикл в фоновом режиме.
</Note>

## Установка

Mellowtel для Windows доступен на NuGet как [`Mellowtel.Win`](https://www.nuget.org/packages/Mellowtel.Win).

### 1. Установите пакет

Из директории вашего проекта:

```bash
dotnet add package Mellowtel.Win
```

Или добавьте `PackageReference` в ваш `.csproj`:

```xml
<PackageReference Include="Mellowtel.Win" Version="1.0.0" />
```

### 2. Добавьте в ваш код

```csharp
using MellowtelWin;

// Инициализируйте Mellowtel с вашим Integration ID
var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
{
    PluginId = "your-app-id"
});

// Если пользователь уже дал согласие, продолжайте без уведомления.
// Если нет, это не будет выполнено и вернет false.
await mellowtel.StartIfOptedInAsync();

// Поток для первого запуска: покажите ваш интерфейс согласия, затем дайте согласие и начните.
if (!mellowtel.GetOptInStatus() && ShowConsentDialog())
{
    mellowtel.OptIn();
    await mellowtel.StartAsync();
}

// При завершении работы приложения
await mellowtel.StopAsync();
mellowtel.Dispose();
```

<Note>
  Замените `YOUR_INTEGRATION_ID` на Integration ID из вашей панели управления Mellowtel и установите `PluginId` как стабильный идентификатор для вашего приложения. `ShowConsentDialog()` — это ваш собственный интерфейс, который возвращает `true` только когда пользователь явно соглашается. См. [Согласие пользователя](#user-consent) ниже.
</Note>

### Опции конфигурации

`MellowtelOptions` позволяет настроить несколько параметров:

| Свойство | Тип | По умолчанию | Описание |
| --- | --- | --- | --- |
| `PluginId` | `string` | `"mellowtel-win"` | Стабильный идентификатор для вашего приложения. Установите что-то уникальное. |
| `MaxDailyRate` | `int` | Встроенное значение по умолчанию | Максимальное количество запросов, которые SDK будет обрабатывать в день. |
| `DisableLogs` | `bool` | `true` | Установите `false`, чтобы включить логи SDK во время разработки. |

## Примеры по типу приложения

<Tabs>
  <Tab title="Консольное приложение">
    ```csharp
    using MellowtelWin;

    class Program
    {
        static async Task Main(string[] args)
        {
            using var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
            {
                PluginId = "your-app-id"
            });

            if (!mellowtel.GetOptInStatus())
            {
                Console.WriteLine("Пользователь должен дать согласие перед использованием этого сервиса.");
                return;
            }

            using var cts = new CancellationTokenSource();
            Console.CancelKeyPress += (s, e) => { e.Cancel = true; cts.Cancel(); };

            try
            {
                await mellowtel.StartAsync(cts.Token);
                Console.WriteLine("Mellowtel запущен. Нажмите Ctrl+C для остановки.");
                await Task.Delay(Timeout.Infinite, cts.Token);
            }
            catch (InvalidOperationException ex)
            {
                Console.WriteLine($"Не удалось запустить: {ex.Message}");
            }
            catch (OperationCanceledException)
            {
                Console.WriteLine("Остановка...");
            }
            finally
            {
                await mellowtel.StopAsync();
            }
        }
    }
    ```
  </Tab>
  <Tab title="WPF приложение">
    Добавьте в ваш `MainWindow.xaml.cs`:

    ```csharp
    using MellowtelWin;
    using System.Windows;

    public partial class MainWindow : Window
    {
        private Mellowtel? _mellowtel;

        public MainWindow()
        {
            InitializeComponent();
            Loaded += Window_Loaded;
        }

        private async void Window_Loaded(object sender, RoutedEventArgs e)
        {
            _mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
            {
                PluginId = "your-app-id"
            });

            _mellowtel.ConnectionStateChanged += (s, connected) =>
            {
                Dispatcher.Invoke(() =>
                {
                    StatusTextBlock.Text = connected ? "Подключено" : "Отключено";
                });
            };

            // Продолжайте без уведомления, если уже дано согласие. В противном случае покажите ваш интерфейс согласия.
            var started = await _mellowtel.StartIfOptedInAsync();
            if (!started)
            {
                // Покажите ваш диалог согласия здесь
            }
        }

        protected override async void OnClosing(CancelEventArgs e)
        {
            if (_mellowtel != null)
            {
                await _mellowtel.StopAsync();
                _mellowtel.Dispose();
            }
            base.OnClosing(e);
        }
    }
    ```

### Наблюдение за состоянием подключения

Экземпляр `Mellowtel` предоставляет событие `ConnectionStateChanged`, которое срабатывает всякий раз, когда WebSocket к бэкенду Mellowtel подключается или отключается. Примеры для WPF и Windows Forms выше подписываются на это событие, чтобы управлять индикатором состояния. Параметр — это `bool`, где `true` означает подключено, а `false` — отключено.

Событие срабатывает в фоновом потоке, поэтому UI-фреймворки, которые требуют привязки потоков, должны передавать обновления в UI-поток. В WPF используйте `Dispatcher.Invoke`, а в Windows Forms — `Control.Invoke` (оба показаны в примерах выше).

<Note>
  **О паттерне WPF `async void OnClosing`.** Пример WPF выше использует `protected override async void OnClosing`, что является простейшей формой, но имеет тонкую проблему: `base.OnClosing(e)` вызывается синхронно, пока `StopAsync()` все еще ожидается, так что окно может закрыться до завершения очистки. Для большинства приложений это нормально, так как процесс сразу же завершает работу. Если вам нужно гарантированное корректное завершение (например, для сброса телеметрии), следуйте [документированному Microsoft паттерну отложенного выполнения](https://learn.microsoft.com/en-us/dotnet/api/system.windows.window.closing), где вы устанавливаете `e.Cancel = true`, `await` для вашей асинхронной работы, а затем явно закрываете окно.
</Note>

## Согласие пользователя

<Warning>
  Отображение диалога согласия **обязательно**. SDK это обеспечивает: `StartAsync()` выбрасывает `InvalidOperationException`, если пользователь не дал согласие.
</Warning>

```csharp
var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
{
    PluginId = "your-app-id"
});

if (!mellowtel.GetOptInStatus())
{
    var userAgreed = ShowConsentDialog(); // ваш собственный диалог

    if (userAgreed)
        mellowtel.OptIn();
    else
        return; // пользователь отказался, не запускайте
}

try
{
    await mellowtel.StartAsync();
}
catch (InvalidOperationException ex)
{
    Console.WriteLine($"Ошибка: {ex.Message}");
}
```

### Что должно включать ваше диалоговое окно согласия

<Steps>
  <Step title="Объясните, что делает Mellowtel">
    Используйте простой язык. Пример: *"Это приложение использует Mellowtel для обмена вашей неиспользованной интернет-полосой. Это позволяет вам [выгода/функция]. Вы можете отказаться в любое время в настройках."*
  </Step>
  <Step title="Предоставьте пользователям четкий выбор">
    Включите отдельные опции Принять и Отклонить.
  </Step>
  <Step title="Ссылка на политику">
    Включите ссылки на [Условия использования](https://www.mellowtel.com/terms-and-conditions) и [Политику конфиденциальности](https://www.mellowtel.com/privacy-policy).
  </Step>
</Steps>

### Позвольте пользователям изменить свое согласие в настройках

```csharp
// Проверьте текущий статус
bool isOptedIn = mellowtel.GetOptInStatus();

// Отказаться
if (userWantsToOptOut)
{
    mellowtel.OptOut();
    await mellowtel.StopAsync();
}

// Снова дать согласие
if (userWantsToOptIn)
{
    mellowtel.OptIn();
    await mellowtel.StartAsync();
}
```

Для более богатого интерфейса (например, отображения, когда пользователь впервые дал согласие) `GetOptInDetails()` возвращает статус согласия вместе с временными метками согласия и отказа:

```csharp
var (isOptedIn, optInDate, optOutDate) = mellowtel.GetOptInDetails();
```

## Устранение неполадок

<AccordionGroup>
  <Accordion title='Ошибка "Пакет не найден"'>
    1. Убедитесь, что ваш проект нацелен на **.NET 10** или выше. `Mellowtel.Win` требует `net10.0`.
    2. Очистите кеш NuGet и восстановите:

    ```bash
    dotnet nuget locals all --clear
    dotnet restore
    ```
  </Accordion>
  <Accordion title='Ошибка "InvalidOperationException: Пользователь не дал согласие"'>
    `StartAsync()` требует явного согласия. Вызовите `OptIn()` после того, как ваш диалог согласия вернет положительный результат, затем вызовите `StartAsync()`. Для тихого возобновления при последующих запусках используйте `StartIfOptedInAsync()`, который не выполняется, если пользователь не дал согласие.
  </Accordion>
</AccordionGroup>
