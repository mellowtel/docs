## Что это такое

Mellowtel не будет делиться ни одним байтом трафика, пока пользователь явно не согласится, и у пользователя всегда должна быть очевидная возможность отказаться. Это жесткое требование SDK и жесткое требование каждой политики магазина браузеров, которая позволяет расширения на основе Mellowtel.

## Почему это важно

Это трафик пользователя. Все, что меньше явного, отзывного согласия, является как неэтичным, так и основанием для удаления из магазина. SDK разработан так, чтобы *легкий* путь также был *соответствующим*. Если ты следуешь рекомендованной реализации, ты не сможешь случайно выпустить версию, которая активируется без согласия.

## Как это работает

При первой установке (и при любом обновлении, которое впервые вводит Mellowtel) ты должен показать пользователю неотключаемый отказ от ответственности, который объясняет, что делает Mellowtel, а затем зафиксировать его выбор. С этого момента ссылка на настройки должна быть доступна из какого-то очевидного места в твоем интерфейсе, чтобы они могли изменить свое мнение.

У тебя есть два варианта реализации:

1. **Рекомендуется:** открыть предварительно созданную страницу включения Mellowtel с помощью `generateAndOpenOptInLink()`. Один вызов метода, полное соответствие, максимизация конверсии включения.
2. **Пользовательский интерфейс:** создать свою собственную поверхность согласия и вызвать `optIn()` / `optOut()` самостоятельно.

## Поверхность API

| Метод | Возвращает | Используй для |
|---|---|---|
| `generateAndOpenOptInLink()` | `Promise<void>` | Открыть размещенную страницу включения (рекомендуемый путь). |
| `getOptInStatus()` | `Promise<boolean>` | Проверить, согласился ли пользователь. |
| `optIn()` | `Promise<void>` | Зафиксировать согласие из твоего интерфейса. |
| `optOut()` | `Promise<void>` | Отозвать согласие из твоего интерфейса. |
| `start()` | `Promise<void>` | Начать обработку запросов после согласия. |
| `generateSettingsLink()` | `Promise<string>` | URL на размещенную страницу настроек, чтобы пользователи могли изменить свой выбор позже. |

Все методы доступны из любой части твоего расширения: всплывающее окно, скрипт содержимого или фоновый скрипт.

## Пользовательский интерфейс

Если ты хочешь полный контроль над поверхностью согласия, зафиксируй выбор пользователя в своем интерфейсе и передай его в SDK:

```javascript
// Пользователь согласился
await mellowtel.optIn();
await mellowtel.start();

// Пользователь отказался или позже отозвал согласие
await mellowtel.optOut();

// Проверка статуса (например, для отображения переключателя)
const hasOptedIn = await mellowtel.getOptInStatus();
```

Какой бы интерфейс ты ни создавал, он должен:

1. Быть неотключаемым при первой установке.
2. По умолчанию быть отключенным, пока пользователь активно не согласится.
3. Быть повторно открываемым из какого-то очевидного места в твоем продукте, чтобы пользователи могли изменить свой выбор.

## Ссылка на настройки

Пользователи всегда должны иметь легкий способ изменить свои настройки. Используй `generateSettingsLink()`, чтобы получить URL на размещенную страницу настроек и разместить его где-то постоянно: в твоем всплывающем окне, на странице опций, на экране учетной записи.

```javascript
const settingsUrl = await mellowtel.generateSettingsLink();
```

## Объявление Mellowtel существующим пользователям

Если ты добавляешь Mellowtel в расширение, у которого уже есть пользователи, ты должен сообщить им, что изменилось, прежде чем запрашивать согласие. Мы предоставляем [шаблон объявления для копирования и вставки](/concepts/update-announcement), который ты можешь адаптировать.
