Здесь вы узнаете, как реализовать библиотеку Mellowtel в вашем проекте расширения для Chrome/Edge.

Оценочное время завершения: 5-7 минут.

# Соответствие политикам браузера

Mellowtel соответствует политикам браузера, если интеграция уважает выбор и настройки пользователей.

Официальные плагины Mellowtel доступны в Chrome и Edge. Скоро мы подадим их и в другие основные браузеры.

Chrome: https://chromewebstore.google.com/detail/support-with-mellowtel/jngbedjpioeongcicaomeideeompcbcc

Edge: https://microsoftedge.microsoft.com/addons/detail/support-with-mellowtel/chpomjnojnimgcaomhmcmfgjinfoekhi

В Chrome некоторые разработчики могут столкнуться с проблемами во время проверки из-за политики единственной цели, которая гласит, что плагин может иметь только одну четко заявленную функцию (но часто интерпретируется непоследовательно в многофункциональных расширениях). 
В таких случаях мы рекомендуем разработчикам делиться своей персональной ссылкой и просить пользователей установить официальный плагин Mellowtel для Chrome, который имеет единственную цель — позволить разработчикам зарабатывать благодаря пользователям, которые хотят поддержать их, делясь своими неиспользуемыми ресурсами.
Разработчики могут создать свою ссылку в [панели управления Mellowtel](https://www.mellowtel.com/dashboard/support-with-mellowtel) и перенаправить пользователей на официальный [плагин Mellowtel для Chrome](https://chromewebstore.google.com/detail/support-with-mellowtel/jngbedjpioeongcicaomeideeompcbcc).

Если у вас есть плагин для Chrome, который работает в области совместного использования полосы пропускания AI, и это единственная цель плагина, вы можете интегрировать Mellowtel непосредственно в ваш плагин.

В то время как Edge, Firefox или Safari не имеют такой политики, поэтому разработчики могут также интегрировать библиотеку непосредственно в свои продукты и убедиться, что они спрашивают своих пользователей, хотят ли они поддержать их или нет, и всегда оставляют им легкий способ изменить свои настройки в любое время из самого плагина.

# Рекомендуемые браузеры

На данный момент мы рекомендуем интегрировать Mellowtel в плагины для Edge, так как мы все еще работаем над оптимизацией версий библиотеки для Firefox и Safari.

# Требования
Перед началом убедитесь, что:

1) Ваш плагин использует [Manifest V3](https://developer.chrome.com/docs/extensions/develop/migrate/what-is-mv3).
2) Ваш код организован в скомпилированном виде с использованием Webpack, Rollup или других сборщиков.
Если у вас не настроен сборщик, сначала следуйте руководству [здесь](/browser-plugins/static-to-webpack)

# Установка npm пакета

Чтобы установить npm пакет, вы можете выполнить следующую команду в вашем терминале:

```npm
npm install mellowtel
```

<Accordion title="Yarn">
    Если вы используете yarn в качестве менеджера пакетов, вы можете выполнить следующую команду:
    ```yarn
    yarn add mellowtel
    ```
</Accordion>

Существует два способа, как вы можете реализовать библиотеку в вашем расширении.

Если вы собираетесь выпустить расширение впервые в Web Store, следуйте Опции 1.
Если у вас уже есть опубликованное расширение в Web Store, прочитайте "Как выбрать?"

### Как выбрать?

Когда вы устанавливаете ваше расширение из Web Store, появляется ли окно предупреждения с просьбой разрешить "Чтение и изменение всех ваших данных на всех веб-сайтах"?

# Опция 1
Если вы выпускаете расширение впервые в Web Store или импортируете библиотеку в расширение, которое запрашивает разрешение на работу на всех веб-сайтах, вы можете следовать Опции 1.

### Настройка вашего манифеста
В вашем файле [manifest.json](https://developer.chrome.com/docs/extensions/reference/manifest) измените секции `permissions` и `host_permissions` следующим образом:

```json
{

  "permissions": [
    "storage",
    "declarativeNetRequest"
  ],
  "host_permissions": [
    "\u003Call_urls\u003E"
  ]

}
```

### Настройка вашего сервисного работника (фонового скрипта)

В вашем файле [service worker](https://developer.chrome.com/docs/extensions/develop/concepts/service-workers) вам нужно импортировать пакет `mellowtel`.
Примечание: Сервисный работник также известен как `background script`.

Вы можете скопировать и вставить следующий код в ваш файл сервисного работника, изменив `configuration_key` на ваш собственный ключ.

```javascript
import Mellowtel from "mellowtel";

let mellowtel;

(async () => {
    mellowtel = new Mellowtel("<configuration_key>"); // Замените на ваш конфигурационный ключ
    await mellowtel.initBackground();
})();
```

<Accordion title="Разбор кода сервисного работника">
Импортируйте пакет mellowtel

```javascript
    import Mellowtel from "mellowtel";
```

Затем вы можете использовать `Mellowtel(configuration_key, options?)` для создания нового экземпляра объекта Mellowtel. Объект Mellowtel является вашей точкой входа в остальную часть SDK Mellowtel. Ваш `configuration_key` необходим при вызове этой функции, так как он идентифицирует ваше расширение для Mellowtel. Вы можете найти ваш `configuration_key` на [панели управления](https://www.mellowtel.com/mellowtel-dashboard/#home)
```javascript
mellowtel = new Mellowtel("<configuration_key>");
```

Инициализируйте Mellowtel, вызвав метод `initBackground`. Этот метод инициализирует объект Mellowtel и настроит необходимые слушатели.
```javascript
await mellowtel.initBackground();
```
</Accordion>

### Настройка вашего скрипта содержимого
Вы также должны импортировать пакет mellowtel в [скрипты содержимого](https://developer.chrome.com/docs/extensions/develop/concepts/content-scripts).

Этот скрипт содержимого должен выполняться в `all_frames` и `<all_urls>` на `document_start`.

Мы предлагаем создать новый скрипт содержимого (например, content_start_mellowtel.js), где вы можете скопировать и вставить следующий код.
Измените `configuration_key` на ваш собственный ключ.

```javascript
import Mellowtel from "mellowtel";

(async () => {
    const mellowtel = new Mellowtel("<configuration_key>"); // Замените на ваш конфигурационный ключ
    await mellowtel.initContentScript();
})();
```

Затем не забудьте также добавить этот скрипт содержимого в ваш файл `manifest.json` в секции content_scripts вместе с любым другим скриптом содержимого, который у вас уже есть.
Вы можете скопировать и вставить следующий код, изменив `name_of_your_content_script.js` на выбранное вами имя.

```json
{
  "content_scripts": [
    {
      "matches": ["<all_urls>"],
      "js": ["<name_of_your_content_script.js>"], // <-- Измените здесь на имя вашего скрипта содержимого
      "run_at": "document_start",
      "all_frames": true
    }
  ]
}
```

### Настройка логики согласия/отказа

Чтобы активировать Mellowtel, пользователи должны явно согласиться на использование сервиса.

Вы можете реализовать логику согласия/отказа двумя способами:
1) Используя рекомендованную реализацию с предварительно созданной страницей, предоставленной Mellowtel.

2) Создавая свои собственные элементы согласия/отказа и используя методы, показанные [здесь](https://docs.mellowtel.com/concepts/opt-in-out#2-custom-elements)

<Info>
Если вы решите создать свои собственные элементы согласия/отказа, рассмотрите возможность использования <a href="/mellowtel-elements/quickstart" target="_blank">**Mellowtel Elements**</a> - предварительно созданные UI-компоненты, которые обрабатывают согласие пользователей и настройки с минимальными усилиями по интеграции.
</Info>

Если вы используете рекомендованную реализацию, откройте ваш файл `service worker` (также известный как фоновый скрипт) и добавьте следующий код:

```javascript
chrome.runtime.onInstalled.addListener(async function(details) {
    console.log("Расширение установлено или обновлено");
    // Если вы хотите обрабатывать первую установку и обновления по-разному
    /**
    if(details.reason === "install"){
        // вызовите функцию для обработки первой установки
    } else if(details.reason === "update") {
        // вызовите функцию для обработки обновления
    }
    **/
    await mellowtel.generateAndOpenOptInLink();
});
```

### Генерация ссылки на настройки

Чтобы дать вашим пользователям полный контроль над их полосой пропускания, необходимо оставить легкодоступный способ для изменения их настроек в любой момент.

Вы можете сгенерировать ссылку, используя метод `generateSettingsLink()`.
Этот метод вернет ссылку, которая откроет страницу настроек Mellowtel, где пользователи могут в любой момент решить, хотят ли они согласиться или отказаться.

Эта ссылка должна быть размещена внутри всплывающего окна, страницы настроек или любого другого места, где пользователи могут легко получить к ней доступ.

Например, вы можете поместить ее в функцию `openSettings()` следующим образом. Вы можете вызвать `openSettings()` при нажатии кнопки или любом другом соответствующем событии:

```javascript
async function openSettings() {
    try {
        // Сгенерируйте и управляйте ссылкой на настройки
        const settingsLink = await mellowtel.generateSettingsLink();
        // Запишите сгенерированную ссылку для отладки
        console.log("Сгенерированная ссылка на настройки:", settingsLink);
        // Теперь вы можете использовать эту ссылку внутри всплывающего окна, страницы настроек или любого элемента интерфейса
    } catch (error) {
        console.error("Ошибка при генерации ссылки на настройки:", error);
    }
}
```
