हम पुराने `configuration_key` को एक नए पहचानकर्ता, जिसे **इंटीग्रेशन आईडी** कहा जाता है, से बदल रहे हैं। यदि आप पहले से ही Mellowtel का उपयोग कर रहे हैं, तो आपको SDK को पास की गई स्ट्रिंग को अपडेट करना होगा।

## संक्षेप में

- अब हर इंटीग्रेशन की अपनी **इंटीग्रेशन आईडी** है, जो `intgr-` से शुरू होती है।
- पुराना `configuration_key` आज भी काम करता है, लेकिन इसे हटा दिया जाएगा।
- केवल कोड परिवर्तन वह स्ट्रिंग है जो `new Mellowtel(...)` को पास की जाती है।

## कॉन्फ़िगरेशन कुंजी बनाम इंटीग्रेशन आईडी

| | कॉन्फ़िगरेशन कुंजी (पुरानी) | इंटीग्रेशन आईडी (नई) |
|---|---|---|
| स्कोप | प्रति खाता एक कुंजी | प्रति इंटीग्रेशन एक आईडी |
| प्रारूप | अस्पष्ट स्ट्रिंग | `intgr-XXXXXXXXXX` |
| आँकड़े | सभी ऐप्स के लिए संयुक्त | प्रति ऐप |

## हम यह परिवर्तन क्यों कर रहे हैं

Mellowtel अब ब्राउज़र एक्सटेंशन्स, डेस्कटॉप ऐप्स, मोबाइल ऐप्स, और वेबसाइट्स में उपलब्ध है। एक साझा कुंजी के साथ, टीमें यह नहीं जान सकती थीं कि कौन सा प्लेटफॉर्म वास्तव में आय ला रहा था।

प्रति ऐप एक इंटीग्रेशन आईडी इसे ठीक करती है। एक बार जब आप माइग्रेट कर लेते हैं, तो आप प्रति ऐप आँकड़े, प्रति ऐप आय, और डैशबोर्ड फीचर्स को अनलॉक कर लेते हैं जिन्हें हम अगला बना रहे हैं ताकि आप प्रत्येक ऐप को स्वतंत्र रूप से बढ़ा सकें।

## आपको क्या बदलने की आवश्यकता है

केवल वह स्ट्रिंग जिसे आप `new Mellowtel(...)` को पास करते हैं।

```javascript
// पहले
const mellowtel = new Mellowtel("<your_configuration_key>");

// बाद में
const mellowtel = new Mellowtel("intgr-MG8tHhnkDN");
```

बाकी सब कुछ वही रहता है।

## अपनी इंटीग्रेशन आईडी प्राप्त करना

1. अपने डैशबोर्ड में [इंटीग्रेशन्स पेज](https://www.mellowtel.com/dashboard/integrations) खोलें।
2. प्रत्येक ऐप के लिए एक इंटीग्रेशन बनाएं, मिलान प्रकार (एक्सटेंशन, डेस्कटॉप, मोबाइल, वेबसाइट) चुनें।
3. इंटीग्रेशन आईडी कॉपी करें और इसे अपने SDK इनिशियलाइज़ेशन में पेस्ट करें।

## पिछड़ी संगतता

मौजूदा कोड काम करता रहेगा। हम पुराने कुंजी को हटाने से पहले पर्याप्त नोटिस के साथ एक समाप्ति तिथि की घोषणा करेंगे। जल्दी माइग्रेट करने से प्रति ऐप फीचर्स अनलॉक होते हैं, इसलिए इंतजार करने का कोई कारण नहीं है।
