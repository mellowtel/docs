## Qué es

Mellowtel no compartirá ni un solo byte de ancho de banda hasta que el usuario haya optado explícitamente, y el usuario siempre debe tener una manera obvia de optar por no participar. Este es un requisito estricto del SDK y un requisito estricto de cada política de tienda de navegadores que permite extensiones basadas en Mellowtel.

## Por qué es importante

Es el ancho de banda del usuario. Cualquier cosa menos que un consentimiento explícito y revocable es tanto poco ético como motivo para la eliminación de la tienda. El SDK está diseñado para que el camino *fácil* sea también el camino *cumplidor*. Si sigues la implementación recomendada, no puedes accidentalmente lanzar una versión que se active sin consentimiento.

## Cómo funciona

En la primera instalación (y en cualquier actualización que introduzca Mellowtel por primera vez) debes mostrar al usuario un aviso no descartable que explique qué hace Mellowtel, y luego capturar su elección. A partir de entonces, un enlace de configuración debe ser accesible desde algún lugar obvio en tu UI para que puedan cambiar de opinión.

Tienes dos opciones de implementación:

1. **Recomendado:** abre la página de opt-in preconstruida de Mellowtel con `generateAndOpenOptInLink()`. Una llamada de método, totalmente cumplidora, maximiza la conversión de opt-in.
2. **UI personalizada:** construye tu propia superficie de consentimiento y llama a `optIn()` / `optOut()` tú mismo.

## Superficie de la API

| Método | Devuelve | Úsalo para |
|---|---|---|
| `generateAndOpenOptInLink()` | `Promise<void>` | Abrir la página de opt-in alojada (camino recomendado). |
| `getOptInStatus()` | `Promise<boolean>` | Verificar si el usuario ya ha optado. |
| `optIn()` | `Promise<void>` | Registrar el consentimiento desde tu propia UI. |
| `optOut()` | `Promise<void>` | Revocar el consentimiento desde tu propia UI. |
| `start()` | `Promise<void>` | Comenzar a atender solicitudes después del opt-in. |
| `generateSettingsLink()` | `Promise<string>` | URL a la página de configuración alojada para que los usuarios puedan cambiar su elección más tarde. |

Todos los métodos son accesibles desde cualquier parte de tu extensión: popup, script de contenido o script de fondo.

## UI personalizada

Si deseas tener control total sobre la superficie de consentimiento, captura la elección del usuario en tu propia UI y reenvíala al SDK:

```javascript
// El usuario aceptó
await mellowtel.optIn();
await mellowtel.start();

// El usuario rechazó o revocó más tarde
await mellowtel.optOut();

// Verificar estado (por ejemplo, para mostrar un interruptor)
const hasOptedIn = await mellowtel.getOptInStatus();
```

Cualquiera que sea la UI que construyas, debe:

1. No ser descartable en la primera instalación.
2. Estar predeterminada a no participar hasta que el usuario acepte activamente.
3. Ser reabrible desde algún lugar obvio en tu producto para que los usuarios puedan cambiar su elección.

## Enlace de configuración

Los usuarios siempre deben tener una manera fácil de cambiar su configuración. Usa `generateSettingsLink()` para obtener una URL a la página de configuración alojada y colócala en algún lugar persistente: tu popup, una página de opciones, una pantalla de cuenta.

```javascript
const settingsUrl = await mellowtel.generateSettingsLink();
```

## Anunciando Mellowtel a usuarios existentes

Si estás agregando Mellowtel a una extensión que ya tiene usuarios, necesitas decirles qué cambió antes de pedir su consentimiento. Proporcionamos una [plantilla de anuncio para copiar y pegar](/concepts/update-announcement) que puedes adaptar.
