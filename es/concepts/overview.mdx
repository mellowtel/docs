Mellowtel se basa en cuatro principios. Léelos una vez y entenderás cada comportamiento, configuración predeterminada y elección de API en el resto de la documentación.

<CardGroup cols={2}>
  <Card title="Opt-in / Opt-out" icon="signature" href="/concepts/opt-in-out">
    Los usuarios deben dar su consentimiento explícito antes de que se comparta cualquier ancho de banda, y pueden revocarlo en cualquier momento.
  </Card>
  <Card title="Privacidad" icon="lock" href="/concepts/privacy">
    Las solicitudes se ejecutan en un entorno aislado y sin credenciales. No se envían cookies, almacenamiento ni datos personales desde el dispositivo.
  </Card>
  <Card title="Limitación de Tasa" icon="stopwatch" href="/concepts/rate-limiting">
    El SDK se regula automáticamente para que la experiencia de navegación del usuario nunca se vea afectada.
  </Card>
  <Card title="Código Abierto" icon="github" href="https://github.com/mellowtel-inc/">
    Cada línea está en GitHub. Si un documento no es claro, el código fuente es la verdad fundamental.
  </Card>
</CardGroup>
