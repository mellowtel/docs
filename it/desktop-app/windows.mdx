Integra Mellowtel nella tua applicazione desktop Windows per consentire agli utenti di condividere la loro larghezza di banda internet inutilizzata in cambio di ricompense o funzionalità premium.

<Warning>
  **Il consenso dell'utente è obbligatorio.** L'SDK genera un `InvalidOperationException` se chiami `StartAsync()` senza che l'utente abbia effettuato l'opt-in.
</Warning>

## Prerequisiti

- Un account Mellowtel e un **ID di integrazione** dal [dashboard](https://www.mellowtel.com/mellowtel-auth/). Ogni app desktop riceve un ID di integrazione unico.
- .NET 10 SDK
- Windows 10 o Windows 11
- Un'applicazione desktop .NET (Console, WPF o Windows Forms)

<Note>
  Vuoi vedere un'integrazione completa e reale prima di integrarla nella tua app? Il campione WPF [mellowtel-pomodoro-windows](https://github.com/mellowtel-inc/mellowtel-pomodoro-windows) mostra l'intero flusso dall'inizio alla fine: dialogo di consenso, attivazione delle impostazioni e ciclo di vita in background.
</Note>

## Installazione

Mellowtel per Windows è disponibile su NuGet come [`Mellowtel.Win`](https://www.nuget.org/packages/Mellowtel.Win).

### 1. Installa il pacchetto

Dalla directory del tuo progetto:

```bash
dotnet add package Mellowtel.Win
```

Oppure aggiungi un `PackageReference` al tuo `.csproj`:

```xml
<PackageReference Include="Mellowtel.Win" Version="1.0.0" />
```

### 2. Aggiungi al tuo codice

```csharp
using MellowtelWin;

// Inizializza Mellowtel con il tuo ID di integrazione
var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
{
    PluginId = "your-app-id"
});

// Se l'utente ha già effettuato l'opt-in, riprendi silenziosamente.
// In caso contrario, questa operazione non ha effetto e restituisce false.
await mellowtel.StartIfOptedInAsync();

// Flusso per la prima volta: mostra la tua UI di consenso, poi effettua l'opt-in e avvia.
if (!mellowtel.GetOptInStatus() && ShowConsentDialog())
{
    mellowtel.OptIn();
    await mellowtel.StartAsync();
}

// Alla chiusura dell'app
await mellowtel.StopAsync();
mellowtel.Dispose();
```

<Note>
  Sostituisci `YOUR_INTEGRATION_ID` con l'ID di integrazione dal tuo dashboard Mellowtel e imposta `PluginId` su un identificatore stabile per la tua app. `ShowConsentDialog()` è la tua UI personalizzata che restituisce `true` solo quando l'utente accetta esplicitamente. Vedi [Consenso dell'utente](#user-consent) qui sotto.
</Note>

### Opzioni di configurazione

`MellowtelOptions` ti consente di regolare alcune cose:

| Proprietà | Tipo | Predefinito | Descrizione |
| --- | --- | --- | --- |
| `PluginId` | `string` | `"mellowtel-win"` | Identificatore stabile per la tua app. Impostalo su qualcosa di unico. |
| `MaxDailyRate` | `int` | Predefinito integrato | Numero massimo di richieste che l'SDK gestirà al giorno. |
| `DisableLogs` | `bool` | `true` | Imposta su `false` per abilitare i log dell'SDK durante lo sviluppo. |

            {
                Console.WriteLine("Arresto in corso...");
            }
            finally
            {
                await mellowtel.StopAsync();
            }
        }
    }

### Osservare lo stato della connessione

L'istanza `Mellowtel` espone un evento `ConnectionStateChanged` che si attiva ogni volta che il WebSocket sottostante al backend di Mellowtel si connette o si disconnette. Gli esempi WPF e Windows Forms sopra si iscrivono a questo evento per gestire un indicatore di stato. Il payload è un `bool` dove `true` significa connesso e `false` significa disconnesso.

L'evento si attiva su un thread in background, quindi i framework UI che applicano l'affinità del thread devono trasferire gli aggiornamenti sul thread UI. In WPF usa `Dispatcher.Invoke`, e in Windows Forms usa `Control.Invoke` (entrambi mostrati negli esempi sopra).

<Note>
  **A proposito del pattern WPF `async void OnClosing`.** L'esempio WPF sopra utilizza `protected override async void OnClosing`, che è la forma più semplice ma ha un problema sottile: `base.OnClosing(e)` viene chiamato in modo sincrono mentre `StopAsync()` è ancora in attesa, quindi la finestra può chiudersi prima che la pulizia sia terminata. Per la maggior parte delle app questo va bene perché il processo termina immediatamente dopo. Se hai bisogno di una chiusura garantita e ordinata (ad esempio, per inviare telemetria), segui il [pattern di differimento documentato da Microsoft](https://learn.microsoft.com/en-us/dotnet/api/system.windows.window.closing) dove imposti `e.Cancel = true`, `await` il tuo lavoro asincrono, e poi chiudi esplicitamente la finestra.
</Note>

## Consenso dell'utente

<Warning>
  Mostrare un dialogo di consenso è **obbligatorio**. L'SDK lo impone: `StartAsync()` genera un `InvalidOperationException` se l'utente non ha effettuato l'opt-in.
</Warning>

```csharp
var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
{
    PluginId = "your-app-id"
});

if (!mellowtel.GetOptInStatus())
{
    var userAgreed = ShowConsentDialog(); // il tuo dialogo personalizzato

    if (userAgreed)
        mellowtel.OptIn();
    else
        return; // l'utente ha rifiutato, non avviare
}

try
{
    await mellowtel.StartAsync();
}
catch (InvalidOperationException ex)
{
    Console.WriteLine($"Errore: {ex.Message}");
}
```

### Cosa deve includere il tuo dialogo di consenso

<Steps>
  <Step title="Spiega cosa fa Mellowtel">
    Usa un linguaggio semplice. Esempio: *"Questa app utilizza Mellowtel per condividere la tua larghezza di banda internet inutilizzata. Questo ti permette di [beneficio/funzionalità]. Puoi disattivare in qualsiasi momento nelle impostazioni."*
  </Step>
  <Step title="Dai agli utenti una scelta chiara">
    Includi opzioni distinte per Accettare e Rifiutare.
  </Step>
  <Step title="Collega alle politiche">
    Includi link ai [Termini di servizio](https://www.mellowtel.com/terms-and-conditions) e alla [Politica sulla privacy](https://www.mellowtel.com/privacy-policy).
  </Step>
</Steps>

### Permetti agli utenti di modificare il loro consenso nelle impostazioni

```csharp
// Controlla lo stato attuale
bool isOptedIn = mellowtel.GetOptInStatus();

// Effettua l'opt-out
if (userWantsToOptOut)
{
    mellowtel.OptOut();
    await mellowtel.StopAsync();
}

// Effettua nuovamente l'opt-in
if (userWantsToOptIn)
{
    mellowtel.OptIn();
    await mellowtel.StartAsync();
}
```

Per una UI più ricca (ad esempio, mostrando quando l'utente ha effettuato l'opt-in per la prima volta), `GetOptInDetails()` restituisce lo stato di opt-in insieme ai timestamp di opt-in e opt-out:

```csharp
var (isOptedIn, optInDate, optOutDate) = mellowtel.GetOptInDetails();
```

## Risoluzione dei problemi

<AccordionGroup>
  <Accordion title='Errore "Pacchetto non trovato"'>
    1. Assicurati che il tuo progetto sia destinato a **.NET 10** o successivo. `Mellowtel.Win` richiede `net10.0`.
    2. Cancella la cache di NuGet e ripristina:

    ```bash
    dotnet nuget locals all --clear
    dotnet restore
    ```
  </Accordion>
  <Accordion title='Errore "InvalidOperationException: User has not opted in"'>
    `StartAsync()` richiede un consenso esplicito. Chiama `OptIn()` dopo che il tuo dialogo di consenso restituisce un risultato positivo, quindi chiama `StartAsync()`. Per un ripristino silenzioso nei lanci successivi, usa `StartIfOptedInAsync()`, che non ha effetto quando l'utente non ha effettuato l'opt-in.
  </Accordion>
</AccordionGroup>
