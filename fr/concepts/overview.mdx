Mellowtel est construit sur quatre principes. Lis-les une fois et tu comprendras chaque comportement, valeur par défaut et choix d’API dans le reste de la documentation.

<CardGroup cols={2}>
  <Card title="Opt-in / Opt-out" icon="signature" href="/concepts/opt-in-out">
    Les utilisateurs doivent donner un consentement explicite avant que toute bande passante soit partagée, et peuvent le révoquer à tout moment.
  </Card>
  <Card title="Confidentialité" icon="lock" href="/concepts/privacy">
    Les requêtes s'exécutent dans un environnement isolé, sans identifiants. Aucun cookie, stockage ou donnée personnelle ne quitte jamais l'appareil.
  </Card>
  <Card title="Limitation de débit" icon="stopwatch" href="/concepts/rate-limiting">
    Le SDK se régule automatiquement pour que l'expérience de navigation de l'utilisateur ne soit jamais affectée.
  </Card>
  <Card title="Open Source" icon="github" href="https://github.com/mellowtel-inc/">
    Chaque ligne est sur GitHub. Si une documentation n'est pas claire, la source est la vérité absolue.
  </Card>
</CardGroup>
