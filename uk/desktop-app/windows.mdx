Інтегруйте Mellowtel у ваш додаток для Windows, щоб дозволити користувачам ділитися невикористаною пропускною здатністю інтернету в обмін на винагороди або преміум-функції.

<Warning>
  **Згода користувача є обов’язковою.** SDK викидає `InvalidOperationException`, якщо ви викликаєте `StartAsync()` без того, щоб користувач погодився.
</Warning>

## Попередні вимоги

- Обліковий запис Mellowtel та **Integration ID** з [панелі керування](https://www.mellowtel.com/mellowtel-auth/). Кожен додаток отримує свій унікальний Integration ID.
- .NET 10 SDK
- Windows 10 або Windows 11
- Додаток на .NET (Console, WPF або Windows Forms)

<Note>
  Хочете побачити повну інтеграцію в реальному світі перед тим, як підключити це до свого додатку? Зразок [mellowtel-pomodoro-windows](https://github.com/mellowtel-inc/mellowtel-pomodoro-windows) для WPF показує повний потік від початку до кінця: діалог згоди, перемикач налаштувань та фоновий життєвий цикл.
</Note>

## Встановлення

Mellowtel для Windows доступний на NuGet як [`Mellowtel.Win`](https://www.nuget.org/packages/Mellowtel.Win).

### 1. Встановіть пакет

З вашої директорії проекту:

```bash
dotnet add package Mellowtel.Win
```

Або додайте `PackageReference` до вашого `.csproj`:

```xml
<PackageReference Include="Mellowtel.Win" Version="1.0.0" />
```

### 2. Додайте до вашого коду

```csharp
using MellowtelWin;

// Ініціалізуйте Mellowtel з вашим Integration ID
var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
{
    PluginId = "your-app-id"
});

// Якщо користувач вже погодився, відновіть роботу без повідомлення.
// Якщо ні, це нічого не робить і повертає false.
await mellowtel.StartIfOptedInAsync();

// Потік для першого запуску: покажіть свій інтерфейс згоди, потім погодьтеся і почніть.
if (!mellowtel.GetOptInStatus() && ShowConsentDialog())
{
    mellowtel.OptIn();
    await mellowtel.StartAsync();
}

// При завершенні роботи додатку
await mellowtel.StopAsync();
mellowtel.Dispose();
```

<Note>
  Замініть `YOUR_INTEGRATION_ID` на Integration ID з вашої панелі керування Mellowtel, і встановіть `PluginId` як стабільний ідентифікатор для вашого додатку. `ShowConsentDialog()` — це ваш власний інтерфейс, який повертає `true` тільки тоді, коли користувач явно погоджується. Дивіться [Згода користувача](#user-consent) нижче.
</Note>

### Опції конфігурації

`MellowtelOptions` дозволяє налаштувати кілька параметрів:

| Властивість | Тип | За замовчуванням | Опис |
| --- | --- | --- | --- |
| `PluginId` | `string` | `"mellowtel-win"` | Стабільний ідентифікатор для вашого додатку. Встановіть це на щось унікальне. |
| `MaxDailyRate` | `int` | Вбудоване значення за замовчуванням | Максимальна кількість запитів, які SDK обробляє за день. |
| `DisableLogs` | `bool` | `true` | Встановіть `false`, щоб увімкнути журнали SDK під час розробки. |

            {
                await mellowtel.StopAsync();

    }

### Спостереження за станом підключення

Екземпляр `Mellowtel` надає подію `ConnectionStateChanged`, яка викликається щоразу, коли підключення WebSocket до бекенду Mellowtel підключається або відключається. Приклади для WPF та Windows Forms вище підписуються на цю подію, щоб керувати індикатором стану. Навантаження — це `bool`, де `true` означає підключено, а `false` означає відключено.

Подія викликається у фоновому потоці, тому UI-фреймворки, які забезпечують прив'язку до потоку, повинні передавати оновлення на UI-потік. У WPF використовуйте `Dispatcher.Invoke`, а в Windows Forms використовуйте `Control.Invoke` (обидва показані у прикладах вище).

<Note>
  **Про WPF `async void OnClosing` патерн.** Приклад WPF вище використовує `protected override async void OnClosing`, що є найпростішою формою, але має тонку проблему: `base.OnClosing(e)` викликається синхронно, поки `StopAsync()` ще очікує, тому вікно може закритися до завершення очищення. Для більшості додатків це нормально, оскільки процес завершується відразу після цього. Якщо вам потрібне гарантоване плавне завершення (наприклад, щоб зберегти телеметрію), дотримуйтесь [документованого Microsoft патерну відкладення](https://learn.microsoft.com/en-us/dotnet/api/system.windows.window.closing), де ви встановлюєте `e.Cancel = true`, `await` вашої асинхронної роботи, а потім явно закриваєте вікно.
</Note>

## Згода користувача

<Warning>
  Відображення діалогу згоди є **обов’язковим**. SDK це забезпечує: `StartAsync()` викидає `InvalidOperationException`, якщо користувач не погодився.
</Warning>

```csharp
var mellowtel = new Mellowtel("YOUR_INTEGRATION_ID", new MellowtelOptions
{
    PluginId = "your-app-id"
});

if (!mellowtel.GetOptInStatus())
{
    var userAgreed = ShowConsentDialog(); // ваш власний діалог

    if (userAgreed)
        mellowtel.OptIn();
    else
        return; // користувач відмовився, не починайте
}

try
{
    await mellowtel.StartAsync();
}
catch (InvalidOperationException ex)
{
    Console.WriteLine($"Помилка: {ex.Message}");
}
```

### Що має включати ваш діалог згоди

<Steps>
  <Step title="Поясніть, що робить Mellowtel">

  </Step>
  <Step title="Надайте користувачам чіткий вибір">
    Включіть окремі варіанти Прийняти та Відхилити.
  </Step>
  <Step title="Посилання на політики">
    Включіть посилання на [Умови використання](https://www.mellowtel.com/terms-and-conditions) та [Політику конфіденційності](https://www.mellowtel.com/privacy-policy).
  </Step>
</Steps>

### Дозвольте користувачам змінювати свою згоду в налаштуваннях

```csharp
// Перевірте поточний статус
bool isOptedIn = mellowtel.GetOptInStatus();

// Відмова
if (userWantsToOptOut)
{
    mellowtel.OptOut();
    await mellowtel.StopAsync();
}

// Повернення згоди
if (userWantsToOptIn)
{
    mellowtel.OptIn();
    await mellowtel.StartAsync();
}
```

Для більш багатого інтерфейсу (наприклад, показуючи, коли користувач вперше погодився), `GetOptInDetails()` повертає статус згоди разом з датами згоди та відмови:

```csharp
var (isOptedIn, optInDate, optOutDate) = mellowtel.GetOptInDetails();
```

## Усунення несправностей

<AccordionGroup>
  <Accordion title='Помилка "Пакет не знайдено"'>
    1. Переконайтеся, що ваш проект націлений на **.NET 10** або пізнішу версію. `Mellowtel.Win` вимагає `net10.0`.
    2. Очистіть кеш NuGet і відновіть:

    ```bash
    dotnet nuget locals all --clear
    dotnet restore
    ```
  </Accordion>
  <Accordion title='Помилка "InvalidOperationException: Користувач не погодився"'>
    `StartAsync()` вимагає явної згоди. Викличте `OptIn()` після того, як ваш діалог згоди поверне позитивний результат, потім викличте `StartAsync()`. Для безмовного відновлення при наступних запусках використовуйте `StartIfOptedInAsync()`, що нічого не робить, коли користувач не погодився.
  </Accordion>
</AccordionGroup>
