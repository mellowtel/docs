Ми замінюємо застарілий `configuration_key` новим ідентифікатором під назвою **Integration ID**. Якщо ви вже використовуєте Mellowtel, вам потрібно буде оновити рядок, який ви передаєте в SDK.

## Коротко

- Кожна інтеграція тепер має свій власний **Integration ID**, з префіксом `intgr-`.
- Старий `configuration_key` все ще працює сьогодні, але буде застарілим.
- Єдина зміна в коді — це рядок, переданий до `new Mellowtel(...)`.

## Ключ конфігурації проти Integration ID

| | Ключ конфігурації (застарілий) | Integration ID (новий) |
|---|---|---|
| Область | Один ключ на акаунт | Один ID на інтеграцію |
| Формат | Непрозорий рядок | `intgr-XXXXXXXXXX` |
| Статистика | Об'єднана для всіх додатків | Для кожного додатка |

## Чому ми робимо цю зміну

Mellowtel тепер використовується в розширеннях для браузерів, настільних додатках, мобільних додатках і на вебсайтах. З одним спільним ключем команди не могли визначити, яка поверхня насправді приносила дохід.

Один Integration ID на додаток виправляє це. Після міграції ви отримаєте доступ до статистики для кожного додатка, доходів для кожного додатка та функцій панелі управління, які ми будуємо, щоб допомогти вам розвивати кожен додаток окремо.

## Що вам потрібно змінити

Тільки рядок, який ви передаєте до `new Mellowtel(...)`.

```javascript
// До
const mellowtel = new Mellowtel("<your_configuration_key>");

// Після
const mellowtel = new Mellowtel("intgr-MG8tHhnkDN");
```

Все інше залишається без змін.

## Отримання вашого Integration ID

1. Відкрийте [сторінку інтеграцій](https://www.mellowtel.com/dashboard/integrations) у вашій панелі управління.
2. Створіть одну інтеграцію на додаток, вибравши відповідний тип (розширення, настільний, мобільний, вебсайт).
3. Скопіюйте Integration ID і вставте його у ваш SDK init.

## Зворотна сумісність

Існуючий код продовжує працювати. Ми оголосимо дату застарівання з достатнім попередженням перед видаленням застарілого ключа. Міграція раніше відкриває функції для кожного додатка, тому немає причин чекати.
