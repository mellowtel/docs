Mellowtel basiert auf vier Prinzipien. Lies diese einmal und du wirst jedes Verhalten, jede Voreinstellung und jede API-Auswahl im Rest der Dokumentation verstehen.

<CardGroup cols={2}>
  <Card title="Opt-in / Opt-out" icon="signature" href="/concepts/opt-in-out">
    Benutzer müssen eine explizite Zustimmung geben, bevor Bandbreite geteilt wird, und können diese jederzeit widerrufen.
  </Card>
  <Card title="Datenschutz" icon="lock" href="/concepts/privacy">
    Anfragen laufen in einer isolierten, anmeldefreien Umgebung. Keine Cookies, Speicher oder persönlichen Daten verlassen jemals das Gerät.
  </Card>
  <Card title="Ratenbegrenzung" icon="stopwatch" href="/concepts/rate-limiting">
    Das SDK drosselt sich automatisch, sodass das Surferlebnis eines Benutzers niemals beeinträchtigt wird.
  </Card>
  <Card title="Open Source" icon="github" href="https://github.com/mellowtel-inc/">
    Jede Zeile ist auf GitHub. Wenn ein Dokument unklar ist, ist der Quellcode die maßgebliche Wahrheit.
  </Card>
</CardGroup>
